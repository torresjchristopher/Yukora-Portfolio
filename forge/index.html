üéØ How It Works

1. Launch Dashboard

$ forge tui

Real-time monitoring of all containers, workflows, and scheduled tasks.

2. Run a Container

$ forge container run python:3.11 python script.py --port 8000:80

Instant execution with port mapping, volume mounting, and resource limits.

3. Schedule Workflows

$ forge workflow run daily_etl
$ forge scheduler schedule daily_etl --cron "0 2 * * *"
$ forge scheduler start

Automatic execution with full DAG support and monitoring.

üìö Technical Architecture

  • Lightweight Isolation: Linux namespaces + Windows Job Objects for process isolation
  • Snapshot-Based Images: Single tar.gz per image, instant extraction, no layering overhead
  • Zero Daemon: Direct execution with no persistent background service
  • Auto-Pruning: Automatic cleanup‚Äîdata never accumulates
  • File-Based State: All configuration in YAML, all data in JSON‚Äîno database required
  • Embedded DAG Engine: Full orchestration built directly into the CLI
  • Real-Time TUI: 2 refreshes/second with <50ms render time
  • Production Benchmarking: Compare against Podman/Docker with scientific rigor
